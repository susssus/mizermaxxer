# Editorial workflow

Guidelines for maintaining bibliographic quality and translation review status on Mizermaxxer.

## Verification tiers

| Status | Meaning | Site display |
|--------|---------|--------------|
| `verified` | Confirmed against primary source or multiple independent references | Green badge |
| `possible` | Strong indirect evidence; details may be incomplete | Amber badge |
| `mention_only` | Known reference without full issue/article detail | Amber badge |
| `needs_verification` | Stub or unconfirmed entry | Red badge |
| `disputed` | Conflicting sources | Red badge |

Promote an issue to `verified` only when you can cite the source in `changelog` or `source_notes`.

## Translation review

Every file in `data/translations/` must include:

- `review_status`: `needs_review` or `reviewed`
- `license`: `CC-BY-NC-4.0` (English translation text)
- `article_id` matching a bibliography article
- `license_notes` crediting scan/source rights where applicable

### Promoting to `reviewed`

1. Read the Japanese source (scan or physical copy).
2. Compare JA / romaji / EN blocks for major omissions or errors.
3. Set `review_status: reviewed` in the translation YAML.
4. Add a changelog entry on the parent issue noting the review date.
5. Run `make build` to refresh `site/src/data/translations.json`.

Flagship magazine interviews and photo spreads should be reviewed before flyer batch translations.

## v1 / v2 entity mapping

- **v1 slugs** (kebab-case) power bibliography, discography, and gigs (`archive.json`).
- **v2 entity IDs** (prefixed snake_case) power `/browse` and `/entity/[id]` (`links_index.json`).
- Canonical venue example: v1 `tokyo-dome` ↔ v2 `venue_tokyo_dome` (see entity file notes).
- `site/src/data/entity_crosswalk.json` is generated by `make build` for cross-link helpers.

## Appearances

`data/appearances/` files are validated by `make entities` but not yet exported to the public catalog. Treat them as research-only until the v2 migration completes.

## Legal posture

- **Project-owned content** (bibliographic YAML, English translations) is released under
  [CC BY-NC 4.0](../LICENSE-CC-BY-NC-4.0.md). Code is [MIT](../LICENSE).
- Mirrored scans are documented in `images/manifest.json` and `/attribution`. When adding new images:

1. Record `source_url` and `source_name` in the manifest.
2. Prefer linking to Cantavanda, Archive.org, or rights-holder archives when mirroring is uncertain.
3. Document decisions in [docs/SOURCES.md](SOURCES.md) — do not assume mirroring is permitted.
